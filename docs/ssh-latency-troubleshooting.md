# SSH 连接境外生产机延迟大 — 问题复盘

## 一、现象
从国内 ssh 到生产机 `prod`（阿里云**雅加达**节点，`8.215.196.203`），终端回显明显卡顿、交互不跟手；每执行一条命令要等 **5～6 秒**。

## 二、根因（三层叠加，逐层揭开）

| 层 | 问题 | 实测数据 |
|---|---|---|
| **1. 无 SSH 连接复用** | 每条命令都重新做一整套 SSH 握手 | 每次建连 **5.9s** |
| **2. 本机代理绕路** | 本机 Clash Verge / Shadowrocket 走 **TUN 模式**，劫持默认路由，把 ssh 流量绕到境外代理节点再回雅加达 | 命令耗时 **1.5s**，全是绕路 |
| **3. 物理 RTT** | 揭开绕路后暴露真实瓶颈：国内宽带→印尼的**直连国际路由质量一般** | 真实 RTT **~450ms**（典型雅加达直连本该 100–150ms）|

## 三、排查方法（可复用）
1. **对照实验定位「握手 vs 链路」**：建一个 ControlMaster 持久连接，对比"建连耗时"和"复用后单命令耗时"——5.9s→1.5s，说明大头是重复握手。
2. **TCP 握手测 RTT**：
   ```bash
   python3 -c "import socket,time;s=time.time();socket.create_connection(('8.215.196.203',22),5).close();print(f'{(time.time()-s)*1000:.0f}ms')"
   ```
3. **⚠️ 关键陷阱**：测出 TCP RTT = **1ms** 是**假象**——TUN 模式代理在本地秒答了 TCP 握手，真实数据往返被代理转发到境外。**别被 TCP RTT 骗，要看应用层（ssh 命令）的真实往返。**
4. **确认代理劫持**：`route -n get default`（看默认路由是否走 `utunX`）+ `ps aux | grep -iE 'clash|surge|sing-box'`。

## 四、解决方案

### ✅ 已实施（收益明确）

#### 1. SSH 连接复用（ControlMaster）
`~/.ssh/config` 的 `Host *` 加：
```
Host *
  ControlMaster auto
  ControlPath ~/.ssh/cm-%C
  ControlPersist 10m
```
- 效果：每条命令 **5.9s → 1.5s**（10 分钟内复用同一连接，免重复握手）。
- `%C` 为 host+port+user 的短哈希，避开 unix socket 路径长度上限。
- 管理已建主连接：`ssh -O check prod`（看状态）、`ssh -O exit prod`（关闭，换网/规则变更后需关掉旧连接以清掉缓存路径）。

#### 2. 代理加目标 IP 直连规则
让 prod 的 IP 走 DIRECT，绕过境外节点绕路：

- **Clash Verge**（Mac）：全局 `Merge.yaml`（全局扩展配置，不被订阅更新覆盖）加：
  ```yaml
  prepend-rules:
    - IP-CIDR,8.215.196.203/32,DIRECT,no-resolve
  ```
  改完需在 GUI 重新应用配置才生效（外部改文件不会自动重载）。
- **Shadowrocket**（手机/iPad）：加规则，类型必须是 **`IP-CIDR`**，动作 `DIRECT`；首页「全局路由」要是「配置/规则」模式。修改本地规则后，回首页把连接开关关掉再打开即重载。
  - **⚠️ 常见错误**：类型选成 `DOMAIN-SUFFIX`。IP 连接没有域名，域名规则永不命中，等于没加。IP 段一律用 `IP-CIDR`。
- 效果：绕路消除，**1.5s → 0.9s**，并暴露出真实 RTT 450ms。

### ⛔ 物理下限（无法再压）
直连后剩余的 ~450ms 是**中国↔印尼国际链路的物理 RTT**，直连已到头。交互式每敲一字符仍需等约一个 RTT 回显，这部分换不掉（除非上专线/中转，成本高收益小）。

> 备注：因国内→印尼**直连**国际路由质量一般（450ms），若机场有**新加坡节点**，经 SG 中转到雅加达**有可能**比直连还快（SG↔雅加达 ~20ms + 中国↔SG ~60-80ms）。与「直连最快」的直觉相反，需实测对比，未深究。

### 🔲 评估后未采用：mosh
mosh 能**本地预测回显**让敲字瞬间跟手（无视 RTT），认证复用 SSH、流量 AES-128-OCB 加密、非 root 运行——安全上不比 SSH 弱。**唯一代价是需对公网放行一段 UDP 端口（60000-61000），扩大攻击面**。生产机安全考量下**决定不上**，维持现状。
（若日后要上：收窄端口 `mosh-server -p 60000:60005` + 安全组只放这几个 UDP + 限源 IP 可把代价降到很低。）

## 五、最终效果
**5.9s/命令 → 0.9s/命令（约 6 倍提升）**。批量操作（`ssh prod '一串命令'`）完全不受回显延迟影响；交互敲命令仍有 ~450ms 物理延迟，可接受。

## 六、一句话结论
> 卡顿 = **重复握手** + **代理绕路** + **物理 RTT** 三者叠加。前两者已通过「SSH 连接复用」和「代理直连规则」消除（6 倍提速）；剩余是中国↔印尼的物理延迟，属硬限制。
