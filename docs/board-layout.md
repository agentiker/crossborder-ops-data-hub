# 运营看板：区块布局与结构

看板单页 `frontend/src/pages/BoardPage.tsx`（约 3000 行单文件），数据来自 `web/routes/board.py` 的
`/board/data`（+ 新品、商品详情、汇率各走独立端点）。本文件记**看板当前的区块组织与布局意图**，
供后续调整时不必重新通读大文件。数据**口径**（GMV/利润/退款/广告怎么算）另见 `docs/business-rules.md`；
演示模块的数据源 backlog 见 `docs/board-data-backlog.md`。

> 移动端是一等公民（PRODUCT.md 原则 4）：所有并列区用 `grid-cols-1 lg:grid-cols-2`，窄屏自动堆叠成多行。

## 页面骨架

顶部固定：标题「运营看板」+ 筛选栏（区域 / 平台 / 店铺〔boss 可切〕/ 日期范围，移动端上划收起）。
下方内容分**两个语义区**，各有区头（`SectionHeader`）：

### 区1「按所选日期」——随上方日期筛选变化

从上到下：

1. **经营概览**（`BusinessOverview`，满宽）
   - KPI 指标行（11 列网格：GMV / 广告消耗 / 订单数 / 销量 / ROAS）
   - 趋势图，2 个 tab：**销售趋势** / **订单·销量（件）**（均真实数据）
   - 广告费构成弹窗（点广告卡 ⓘ 打开）
2. **预估利润 | 渠道分布（GMV 占比）**（`ProfitCard` | `ChannelPie`，桌面并列两列 / 移动端两行）
3. **爆款商品**（`HotProducts`，满宽）——2 tab：按销量 / 按 GMV；点商品行开详情弹窗（大图 + SKU 占比 + 渠道 4 分）
4. **退款 / 取消**（`RefundSection`，满宽）——退款额/率/单数/流失 + 退款趋势图 + 取消构成三分堆叠条

### 区2「实时·固定口径」——不随日期筛选变化

1. **库存健康 | 费率监控**（`InventoryHealth` | `FeeRateMonitor`，桌面并列 / 移动端两行）
   - 库存健康 2 tab：汇总（仪表盘 + 风险环图）/ 商品明细（搜索 + 排序表，移动端切卡片式 + 分页）
   - 费率监控：三态徽章（正常/异常/积累中）+ 当前 vs 基准费率 + 趋势 + 异常时归因拆分
2. **近 N 天新品**（`NewProducts`，满宽，独立端点 `/board/new-products`）——近 60 天上线商品 + 每日销量曲线 + 爆单判定
3. **待发货履约**（`OrderSection`，满宽）——待发货分桶（超时/临界/正常）+ 明细表 + 分页

浮层：`AskAiSheet`（点任意「问 AI」入口）、`ImageViewer`（点商品小图看大图）、各弹窗。

## 设计意图（改动前先读，别推翻既有决策）

- **两区语义分层**：区1 是「这段时间生意怎么样」（随日期变），区2 是「此刻要处置什么」（当前快照/固定窗口）。
  一个模块放哪个区，看它的口径是否随日期筛选变化——放错区 = 口径错位（退款曾误置区2，2026-07-04 归位区1）。
- **满宽 vs 并列**：信息量饱满的大卡（爆款榜、退款趋势、待发货表）优先满宽，保证图表/表格可读；
  偏瘦的卡才并列（利润卡纵向堆叠、渠道环图固定高，二者并列可接受）。两个胖卡别硬塞一行。
- **不做演示数据**：后端没有真实数据源的维度，**宁可不做也不放占位**（守 PRODUCT.md「数据可信即设计」）。
  2026-07-04 已下线全部演示 tab。将来数据源就绪走「新建真实卡」而非「先摆演示占位」。
- **口径透明**：数字旁的 InfoTooltip / 「问 AI」入口是体验的一部分（利润 GMV 与概览 GMV 口径不同等，
  靠 tooltip 白话解释而非改数字）。
